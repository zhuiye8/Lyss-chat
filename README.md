## 项目概述

Lyss-chat 2.0 是对原有 Lyss-chat 项目的全面重构，旨在解决性能瓶颈、用户体验不佳以及多处 bug 等问题。新版本采用 React 18、Ant Design X（AI 交互界面）和 Ant Design 5.x（通用 UI 组件）作为前端技术栈，Go 和 cloudwego/Eino 作为后端技术栈，实现更高效的系统架构，提供更流畅的用户体验。前端的 Ant Design X 与 Ant Design 的关系类似于后端中 EINO 与 gorilla/mux 的关系，一个专注于特定领域，一个提供通用功能。

### 项目目标

1. 提升系统性能，支持更大规模的并发用户
2. 优化用户体验，参考主流 AI 聊天产品的设计
3. 增强系统安全性，特别是 API 密钥的管理
4. 改进模型管理架构，实现更灵活的模型分发机制
5. 实现更细粒度的权限控制系统
6. 提供更完善的代码画布功能
7. 支持容器化部署，简化运维流程

## 技术栈

### 前端技术

- **核心框架**：React 18
- **UI 组件库**：
  - **Ant Design X**：专为 AI 应用设计的组件库，负责 AI 交互相关的 UI
  - **Ant Design 5.x**：成熟的 React UI 组件库，处理通用 UI 需求
- **开发语言**：TypeScript
- **状态管理**：Zustand
- **路由管理**：React Router
- **HTTP 客户端**：Axios
- **表单处理**：React Hook Form + Zod
- **代码编辑器**：Monaco Editor
- **构建工具**：Vite

### 后端技术

- **开发语言**：Go
- **Web 框架**：cloudwego/Eino
- **API 风格**：RESTful
- **认证机制**：JWT
- **文档工具**：Swagger/OpenAPI

### 数据库与存储

- **主数据库**：PostgreSQL（关系型数据，强一致性需求）
- **缓存系统**：Redis（会话管理、频繁访问数据）
- **对象存储**：MinIO（文件、图片存储）

### 部署与运维

- **容器化**：Docker
- **编排工具**：Docker Compose（开发环境）/ Kubernetes（生产环境）
- **CI/CD**：GitHub Actions
- **监控系统**：Prometheus + Grafana

## 系统架构

Lyss-chat 2.0 采用前后端分离的微服务架构，主要包括以下几个部分：

1. **前端应用**：React 单页应用，负责用户界面和交互
2. **API 网关**：统一入口，负责请求路由、认证和限流
3. **核心服务**：
   - 用户服务：用户管理、认证和授权
   - 模型服务：模型管理、API 密钥管理
   - 聊天服务：对话管理、消息处理
   - 代码服务：代码画布、代码执行
   - 管理服务：系统配置、统计分析
4. **数据存储**：
   - PostgreSQL：结构化数据存储
   - Redis：缓存和会话管理
   - MinIO：文件和图片存储
