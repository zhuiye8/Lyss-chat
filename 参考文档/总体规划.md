## 项目概述

Lyss-chat 2.0 是对原有 Lyss-chat 项目的全面重构，旨在解决性能瓶颈、用户体验不佳以及多处 bug 等问题。新版本采用 React 18、Ant Design X（AI 交互界面）和 Ant Design 5.x（通用 UI 组件）作为前端技术栈，Go 和 cloudwego/Eino 作为后端技术栈，实现更高效的系统架构，提供更流畅的用户体验。前端的 Ant Design X 与 Ant Design 的关系类似于后端中 EINO 与 gorilla/mux 的关系，一个专注于特定领域，一个提供通用功能。

### 项目目标

1. 提升系统性能，支持更大规模的并发用户
2. 优化用户体验，参考主流 AI 聊天产品的设计
3. 增强系统安全性，特别是 API 密钥的管理
4. 改进模型管理架构，实现更灵活的模型分发机制
5. 实现更细粒度的权限控制系统
6. 提供更完善的代码画布功能
7. 支持容器化部署，简化运维流程

## 技术栈选型

### 前端技术

- **核心框架**：React 18
- **UI 组件库**：
  - **Ant Design X**：专为 AI 应用设计的组件库，负责 AI 交互相关的 UI
  - **Ant Design 5.x**：成熟的 React UI 组件库，处理通用 UI 需求
  - 这种组合类似于后端中 EINO（专用 AI 引擎）与 gorilla/mux（通用 Web 框架）的关系
- **开发语言**：TypeScript
- **状态管理**：Zustand
- **路由管理**：React Router
- **HTTP 客户端**：Axios
- **表单处理**：React Hook Form + Zod
- **代码编辑器**：Monaco Editor
- **构建工具**：Vite

### 后端技术

- **开发语言**：Go
- **Web 框架**：cloudwego/Eino
- **API 风格**：RESTful
- **认证机制**：JWT
- **文档工具**：Swagger/OpenAPI

### 数据库与存储

- **主数据库**：PostgreSQL（关系型数据，强一致性需求）
- **缓存系统**：Redis（会话管理、频繁访问数据）
- **对象存储**：MinIO（文件、图片存储）

### 部署与运维

- **容器化**：Docker
- **编排工具**：Docker Compose（开发环境）/ Kubernetes（生产环境）
- **CI/CD**：GitHub Actions
- **监控系统**：Prometheus + Grafana

## 系统架构

Lyss-chat 2.0 采用前后端分离的微服务架构，主要包括以下几个部分：

1. **前端应用**：React 单页应用，负责用户界面和交互
2. **API 网关**：统一入口，负责请求路由、认证和限流
3. **核心服务**：
   - 用户服务：用户管理、认证和授权
   - 模型服务：模型管理、API 密钥管理
   - 聊天服务：对话管理、消息处理
   - 代码服务：代码画布、代码执行
   - 管理服务：系统配置、统计分析
4. **数据存储**：
   - PostgreSQL：结构化数据存储
   - Redis：缓存和会话管理
   - MinIO：文件和图片存储

详细的系统架构图和组件交互流程请参考[系统架构设计](./系统架构设计.md)文档。

## 核心模块

### 1. 用户认证模块

- 用户注册、登录和密码管理
- 基于 JWT 的认证机制
- 基于 RBAC 的权限控制
- 多租户支持

### 2. 模型管理模块

- 模型提供商管理
- 模型配置和参数管理
- API 密钥安全存储
- 模型分发和访问控制

### 3. 聊天引擎模块

- 对话管理
- 消息处理和流式响应
- 多模态输入（文本、图像、文件）
- 上下文管理和历史记录

### 4. 代码画布模块

- 代码编辑器集成
- 语法高亮和自动完成
- 代码执行环境
- 代码分享和导出

### 5. 管理后台模块

- 用户和角色管理
- 系统配置管理
- 使用统计和分析
- 日志和审计

每个模块的详细设计请参考对应的模块设计文档。

## 开发阶段

项目开发分为以下几个阶段：

### 阶段一：基础设施搭建（4周）

- 技术栈选型确认
- 开发环境搭建
- 基础框架实现
- 数据库设计和实现

### 阶段二：核心功能开发（8周）

- 用户认证模块实现
- 模型管理模块实现
- 聊天引擎模块实现
- 基础 UI 实现

### 阶段三：高级功能开发（6周）

- 代码画布模块实现
- 管理后台模块实现
- 高级 UI 组件实现
- 性能优化

### 阶段四：测试与部署（4周）

- 单元测试和集成测试
- 性能测试和安全测试
- 容器化配置
- 部署流程实现

### 阶段五：文档与上线（2周）

- 用户文档编写
- 开发文档完善
- 系统上线
- 用户培训
